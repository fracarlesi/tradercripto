# MCP Server Configuration & Best Practices

This document describes the Model Context Protocol (MCP) servers configuration and usage guidelines for this project.

## Active MCP Servers

### 1. claude-context (Semantic Code Search)
**Purpose**: Semantic code search and navigation using OpenAI embeddings + AST-based chunking

**Stats**:
- 153 files indexed
- 1229 code chunks
- AST splitter with automatic fallback
- OpenAI text-embedding-3-small embeddings

**Critical Usage Rules**:
1. **ALWAYS re-index BEFORE searching** - Index does NOT auto-update!
2. **Mandatory 15-second wait** after indexing before searching
3. Use `force=true` to ensure fresh index

```python
# CORRECT workflow - DO THIS EVERY TIME
mcp__claude-context__index_codebase(
    path="/absolute/path/to/project",
    splitter="ast",
    force=true  # Always force for fresh index
)

# WAIT 15 seconds!
Bash(command="sleep 15", description="Wait for indexing")

# Then search
mcp__claude-context__search_code(
    path="/absolute/path/to/project",
    query="semantic search query",
    limit=5
)
```

**When to Use**:
- Finding code by semantic meaning (not exact string match)
- Understanding code architecture and patterns
- Locating similar implementations
- Answering "how does X work?" questions

**Performance**: ~10-15 seconds for full re-index, worth the accuracy

### 2. context7 (Library Documentation)
**Purpose**: Fetch up-to-date documentation for any external library

**Usage**:
```python
# Step 1: Resolve library name to ID
mcp__context7__resolve-library-id(
    libraryName="fastapi"
)

# Step 2: Fetch docs
mcp__context7__get-library-docs(
    context7CompatibleLibraryID="/tiangolo/fastapi",
    topic="async routes",  # Optional
    tokens=5000  # Default, increase if needed
)
```

**When to Use**:
- Need latest documentation for external libraries
- Understanding library-specific patterns
- Debugging library integration issues
- Before implementing features with external dependencies

## General MCP Best Practices

### 1. Always Use Absolute Paths
```python
# CORRECT
path="/Users/user/project"

# WRONG
path="./backend"
```

### 2. Index-Search-Implement Workflow
```
1. Re-index with force=true
2. Wait 15 seconds
3. Semantic search for relevant code
4. Read specific files identified
5. Implement changes
6. Re-index for next search
```

### 3. Combine MCP Servers Strategically
- **claude-context**: Find internal code patterns
- **context7**: Learn external library usage
- **perplexity-ask**: Research best practices
- **playwright**: Validate implementation

### 4. Error Handling
If MCP tool fails:
1. Check absolute path is correct
2. Verify server is running (`/bashes` command)
3. Check network connectivity (for web-based servers)
4. Re-index if search returns stale results

## Project-Specific Configuration

### Code Indexing Strategy
- **Frequency**: Re-index after major refactoring
- **Splitter**: AST for Python (automatic fallback for other files)
- **Custom extensions**: None (use defaults)
- **Ignore patterns**: Default (node_modules, .git, etc.)

### Search Optimization
- **Limit**: Start with 5-10 results
- **Extension filter**: Use `.py` for Python-only searches
- **Query style**: Natural language, focus on intent not keywords

## Troubleshooting

### Issue: "Codebase not indexed"
**Solution**: Run index_codebase with force=true, wait 15 seconds

### Issue: Search returns outdated results
**Solution**: Re-index before searching (index doesn't auto-update)

### Issue: Search returns no results
**Solution**:
1. Verify path is absolute
2. Check query is semantic (not regex/exact match)
3. Try broader query
4. Increase result limit

### Issue: Context7 library not found
**Solution**: Use resolve-library-id first, check exact library name spelling
